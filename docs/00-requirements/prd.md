# 产品需求文档 (PRD)

## 1. 项目概述

### 1.1 项目名称
BinanceSquareBot - 币安广场自动推文机器人

### 1.2 项目背景
币安广场是币安的社交媒体平台，需要自动将Fn新闻转化为符合格式要求的推文并定时发布。手动操作耗时，需要自动化。

### 1.3 核心目标
每小时自动爬取最新Fn新闻，通过LLM生成符合规范的币安广场推文，并自动发布。

### 1.4 目标用户
加密货币交易者、内容创作者、需要定时更新币安广场的运营人员。

---

## 2. 核心功能需求

### 2.1 需求清单

| 需求ID | 需求名称 | 需求描述 | 优先级 |
|--------|----------|----------|--------|
| **REQ-BOT-001** | 定时增量爬取Fn新闻 | 每小时从Fn网站爬取新闻列表，使用SQLite存储已处理文章URL的MD5进行去重，只处理新文章 | P0 |
| **REQ-BOT-002** | 多API密钥轮询 | 从环境变量读取多个API密钥（逗号分隔），轮询使用每个密钥发布推文 | P0 |
| **REQ-BOT-003** | LLM推文生成 | 基于新闻内容，使用LangGraph + LangChain调用LLM生成符合格式要求的币安广场推文 | P0 |
| **REQ-BOT-004** | 推文格式强制校验 | 生成后必须校验：100<字符数<800，话题#数量≤2，代币$数量≤2，不满足则重试生成 | P0 |
| **REQ-BOT-005** | 币安广场API发布 | 调用币安广场API发布生成好的推文 | P0 |
| **REQ-BOT-006** | CLI命令行接口 | 使用Typer提供CLI命令，支持手动触发运行 | P1 |
| **REQ-BOT-007** | GitHub Actions调度 | 配置GitHub Actions每小时自动运行一次 | P1 |

## 3. 推文格式约束（必须严格遵守）

| 约束项 | 约束值 | 说明 |
|--------|--------|------|
| 最小字符数 | 101 | 必须大于100字符 |
| 最大字符数 | 799 | 必须小于800字符 |
| 话题标签(`#`)最大数量 | 2 | 包括话题标签的#符号，总数量小于3 → ≤2 |
| 代币标签(`$`)最大数量 | 2 | 包括代币标签的$符号，总数量小于3 → ≤2 |

## 4. 处理流程

```
1. 启动（CLI命令或GitHub Actions）
   ↓
2. 爬取Fn新闻列表 → 获取最新新闻
   ↓
3. 计算每条新闻URL的MD5 → 查询SQLite
   ↓
4. 如果已存在 → 跳过（去重）
   如果不存在 → 保存MD5到SQLite → 继续处理
   ↓
5. 遍历每个API密钥
   ↓
6. 遍历每条新新闻
   ↓
7. 调用LLM（LangGraph工作流）生成推文
   ↓
8. 校验推文格式 → 不满足则重试（最大重试N次）
   ↓
9. 格式校验通过 → 调用币安广场API发布
   ↓
10. 完成 → 退出
```

## 5. 非功能需求

### 5.1 性能需求
- 单次爬取：新闻列表通常在10-50条，处理时间<5分钟
- 每小时执行一次，GitHub Actions免费配额足够

### 5.2 安全需求
- API密钥通过环境变量传入，不写入代码
- .gitignore必须包含 `.env`
- API密钥不记录到日志

### 5.3 可维护性
- 代码模块化，易于修改API端点和Prompt
- 单元测试覆盖核心逻辑（去重、格式校验、生成流程）
