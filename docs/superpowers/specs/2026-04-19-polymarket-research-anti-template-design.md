# Polymarket 研报去模板化优化设计

> 问题：当前固定章节结构让每篇研报都感觉是模板输出，缺乏自然变化

## 目标

修改 `research_generator.py` 中的 Prompt，移除强制固定章节顺序约束，让 LLM 可以根据不同市场自由组织文章结构，产生更多样化、更自然的输出，同时保留质量约束（字数、话题标签等）。

## 背景

- 当前 `build_prompt()` 方法强制要求：`内容结构（严格按此顺序）` 列出 7 个固定章节
- 每个章节都有明确的内容要求
- 这导致 LLM 每次输出都严格遵循相同结构，显得机械化

## 设计方案

### 修改原则

| 项目 | 处理 |
|------|------|
| 核心写作目标（抓眼球、错配分析、引发讨论） | **保留** |
| 格式约束（字数、#标签、$标签） | **保留** |
| 写作风格要求（口语化、短句、犀利自信） | **保留** |
| 禁止事项列表 | **保留** |
| 严格固定顺序的 7 步章节结构 | **移除强制约束** |
| 改为 | 内容要点建议，自由排序组织 |

### Prompt 变化

**修改前：**
```
内容结构（严格按此顺序）
爆炸式开头（前 20 字必须抓住眼球）
必须用 "市场疯了！"、"所有人都错了！"、"这个概率太离谱了！"、"我刚梭了 5000USDC 赌它反转" 这类强情绪开头
立刻抛出你的反常识核心观点，不要铺垫
第一句就点明当前市场概率有多荒谬
事件极简速览
用 1-2 句话说清楚这个预测市场到底在赌什么
... (共 7 个固定章节)
```

**修改后：**
```
内容要点建议（你可以自由排序和组织，不必拘泥固定顺序）
- 开篇必须抓住眼球：用强情绪第一句直接点明核心观点，不要铺垫
- 说清楚这个预测市场到底在赌什么事件
- 分析当前 {market.yes_price:.1%} 的 YES 概率反映了什么样的市场共识
- 指出市场正在犯的错误，给出你的独家分析（这是核心价值）
- 给读者一些交易策略参考建议
- 结尾用一个争议性问题引发互动讨论
- 文末加上免责声明

根据市场的具体情况，你可以调整内容顺序、合并或拆分段落，怎么自然怎么来。
```

### 为什么这个方案

1. **自由度足够**：LLM 可以根据不同问题类型选择最合适的叙事结构
   - 事件清晰的问题可以先给结论
   - 复杂背景的问题可以先介绍事件
   - 争议性大的问题可以直接展开分析

2. **质量可控**：保留了所有硬性要求（字数、标签、风格），不会跑题

3. **改动最小**：只修改 Prompt，不需要改动代码结构，测试成本低

## 受影响文件

- `src/binance_square_bot/services/research_generator.py` - 修改 `build_prompt()` 方法中的 Prompt 文本
- 测试不需要新增，现有测试可以继续通过

## 验收标准

- LLM 生成的文章结构多样化，不再每篇都是相同顺序
- 仍然满足格式验证（字数、标签数量等）
- 所有现有测试通过
